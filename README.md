# dp-proj-00-02

Aplicación React Router (Remix) con Firebase: autenticación (Auth) y Firestore. Réplica de funcionalidad del sistema (usuarios y roles) con layout tipo panel, temas claro/oscuro y mantenimientos.

[![Open in StackBlitz](https://developer.stackblitz.com/img/open_in_stackblitz.svg)](https://stackblitz.com/github/remix-run/react-router-templates/tree/main/default)

## Features

- 🚀 Server-side rendering
- ⚡️ Hot Module Replacement (HMR)
- 📦 Asset bundling and optimization
- 🔄 Data loading and mutations
- 🔒 TypeScript by default
- 🎉 TailwindCSS for styling
- 📖 [React Router docs](https://reactrouter.com/)

## Getting Started

### Installation

Install the dependencies:

```bash
npm install
```

### Development

Start the development server with HMR:

```bash
npm run dev
```

Your application will be available at `http://localhost:5173` (o el puerto que indique Vite).

### Firebase

1. Crea un proyecto en [Firebase Console](https://console.firebase.google.com/) y activa **Authentication** (Email/Password) y **Firestore**.
2. Copia `.env.example` a `.env` y rellena las variables con la configuración de tu app web en Firebase:
   - `VITE_FIREBASE_API_KEY`
   - `VITE_FIREBASE_AUTH_DOMAIN`
   - `VITE_FIREBASE_PROJECT_ID`
   - `VITE_FIREBASE_STORAGE_BUCKET`
   - `VITE_FIREBASE_MESSAGING_SENDER_ID`
   - `VITE_FIREBASE_APP_ID`
3. En Firestore, crea la colección `roles` con al menos un documento (ej. `admin` con campos `name` y `description`). La colección `profiles` se crea al registrarse usuarios.
4. Opcional: despliega reglas de seguridad con `firestore.rules.example` como referencia (cópialo a `firestore.rules` y `firebase deploy --only firestore:rules`).

## Deployment

### Docker Deployment

To build and run using Docker:

```bash
docker build -t my-app .

# Run the container
docker run -p 3000:3000 my-app
```

The containerized application can be deployed to any platform that supports Docker, including:

- AWS ECS
- Google Cloud Run
- Azure Container Apps
- Digital Ocean App Platform
- Fly.io
- Railway

### DIY Deployment

If you're familiar with deploying Node applications, the built-in app server is production-ready.

Make sure to deploy the output of `npm run build`

```
├── package.json
├── package-lock.json (or pnpm-lock.yaml, or bun.lockb)
├── build/
│   ├── client/    # Static assets
│   └── server/    # Server-side code
```
